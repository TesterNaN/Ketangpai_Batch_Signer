# 课堂派批量签到助手 (Ketangpai Batch Signer)

一个用于自动扫描课堂派签到二维码并完成批量签到的Android应用。

## 功能描述

*   **二维码扫描**：使用设备摄像头扫描课堂派发布的签到二维码，自动识别并提取签到链接。
*   **自动批量签到**：扫描成功后，应用自动读取已保存的账号列表，依次执行登录和签到操作。
*   **账号管理**：
    *   添加：通过界面手动输入课堂派账号（用户名/手机号）和密码。
    *   存储：使用`EncryptedSharedPreferences`安全地加密存储账号信息。
    *   删除：支持从列表中移除已保存的账号。
*   **结果反馈**：清晰展示每个账号的签到状态（成功/失败），并在主界面汇总签到结果。
*   **深色模式**：应用界面支持跟随系统设置自动切换浅色/深色主题。

## 技术实现

*   **架构**：原生Android开发 (Java)
*   **二维码扫描**：集成 `jenly1314/WeChatQRCode` 库，基于OpenCV实现高效识别。
*   **网络请求**：使用 `OkHttp3` 处理与课堂派服务器的登录及签到API通信。
*   **数据存储**：使用 `EncryptedSharedPreferences` (AndroidX Security) 安全存储用户账号。
*   **权限管理**：使用 `PermissionX` 优雅地处理运行时相机权限申请。
*   **界面**：基于 `Material Components`，支持深色模式。

## 配置与运行

### 环境要求
*   Android SDK
*   Android Studio Giraffe 或更高版本
*   `compileSdkVersion` 34
*   一部具有摄像头的Android手机

### 关键配置步骤
1.  **模型文件**：本项目已将 `WeChatQRCode` 库运行所需的模型文件 (`detect.prototxt`, `detect.caffemodel`, `sr.prototxt`, `sr.caffemodel`) 内置在 `app/src/main/assets/models/` 目录下。**无需额外下载或放置，项目已包含。**
2.  **依赖与同步**：项目的所有依赖已在 `libs.versions.toml` 中统一定义。在 Android Studio 中打开项目，Gradle 将自动同步并下载所需库。
3.  **初始化**：应用启动时 (`App` 类) 会自动完成 `OpenCV` 与 `WeChatQRCodeDetector` 的初始化，扫码功能即可直接使用。

## 使用说明

1.  **首次使用**：启动应用后，点击“添加”按钮，输入你的课堂派账号和密码。
2.  **开始签到**：点击“扫描二维码”按钮，授予相机权限后，扫描课堂派老师提供的签到二维码。
3.  **等待结果**：扫描成功后，应用会自动开始批量签到，并在界面中显示每个账号的签到进度和最终结果。

## 注意事项

*   请确保输入的课堂派账号密码正确。
*   使用本应用需遵守课堂派的相关用户协议及你所在学校的规章制度。
*   签到成功率受网络状态和课堂派服务器响应影响。
*   账号密码仅加密存储于本地设备，开发者无法获取。
